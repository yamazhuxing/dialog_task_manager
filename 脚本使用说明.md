# 阿里脚本使用说明
---
## 📋 目录概览（2026.07.06 更新）
当前目录共4个核心脚本，覆盖**格式转换→质检→难度评估**全流程：
| 脚本文件名 | 功能定位 | 所属环节 |
|-----------|----------|----------|
| `convert_hermes.py` | Hermes格式转换工具 | 前置处理：将Hermes私有格式转为Anthropic标准Call格式 |
| `convert_openclaw.py` | OpenClaw格式转换工具 | 前置处理：将OpenClaw会话格式转为Anthropic标准Call格式 |
| `quality_check.py` | Claude模型轨迹质检工具 | 质检环节：校验转换后的session轨迹是否符合质量要求 |
| `batch_deepseek_simple.py` | 极简版难度评估工具 | 评估环节：批量调用DeepSeek-V4-Pro对合格轨迹进行难度标注 |

## 🔧 各脚本详细说明
---
### 1. `convert_hermes.py` - Hermes格式转换工具
#### 功能描述
将Hermes平台导出的私有JSON格式会话数据，转换为Anthropic原生Call-level格式，是后续质检和评估的前置步骤。
#### 核心功能
- ✅ 保留完整thinking block、signature、tool_use/tool_result原始结构
- ✅ 自动按session拆分，每个session生成独立文件夹，每个call生成独立json文件
- ✅ 自动标准化模型名称，兼容opus 4.6/4.7/4.8版本
- ✅ 自动补全缺失的字段，保证格式符合Anthropic规范
#### 用法示例
```bash
python convert_hermes.py --input_dir "D:\sessions\test\hermes原始数据目录"
```
#### 参数说明
| 参数 | 必填 | 说明 |
|------|------|------|
| `--input_dir` | 是 | Hermes原始JSON文件所在目录 |
| `--output_dir` | 否 | 转换后输出目录，默认自动生成：`输入目录名-待质检数据-YYYYMMDD` |
---
### 2. `convert_openclaw.py` - OpenClaw格式转换工具
#### 功能描述
将OpenClaw平台导出的会话格式数据，转换为Anthropic原生Call-level格式，适配OpenClaw导出的数据结构。
#### 核心功能
- ✅ 自动解析OpenClaw会话日志，拆分每次LLM call记录
- ✅ 保留完整的工具调用、思考过程、回复内容结构
- ✅ 自动生成符合Anthropic规范的request/response结构
- ✅ 自动处理跨轮上下文拼接，保证call的完整性
#### 用法示例
```bash
python convert_openclaw.py --input_dir "D:\sessions\test\openclaw原始数据目录"
```
#### 参数说明
| 参数 | 必填 | 说明 |
|------|------|------|
| `--input_dir` | 是 | OpenClaw原始会话文件所在目录 |
| `--output_dir` | 否 | 转换后输出目录，默认同输入目录同级生成`-待质检数据`后缀目录 |
---
### 3. `quality_check.py` - 轨迹质检脚本
#### 功能描述
对转换后的Anthropic格式session轨迹进行多维度质量检查，自动筛选出符合要求的合格数据，淘汰低质量、格式错误、不完整的轨迹。
#### 核心功能
- ✅ 支持**固定时间筛选**：脚本顶部`DEFAULT_AFTER_TIME`变量可写死时间，自动过滤早于指定时间的旧轨迹
- ✅ 支持命令行临时覆盖时间：`--after`参数优先级高于固定配置，格式支持`YYYY-MM-DD`/`YYYY-MM-DD HH:MM:SS`
- ✅ 30+项质检规则：覆盖session一致性、格式正确性、内容质量、完整性、低质量对话过滤等多维度
- ✅ 自动分类输出：合格数据存入`pass`目录，不合格存入`fail`目录+错误记录CSV，质检报告存入`report`目录
#### 用法示例
##### 基础用法（使用脚本固定时间筛选）
```bash
python quality_check.py --input_dir "D:\sessions\test\待质检目录"
```
##### 临时指定筛选时间
```bash
python quality_check.py --input_dir "D:\sessions\test\待质检目录" --after "2026-07-01"
```
##### 禁用时间筛选
修改脚本顶部`DEFAULT_AFTER_TIME = None`后运行：
```bash
python quality_check.py --input_dir "D:\sessions\test\待质检目录"
```
---
### 4. `batch_deepseek_simple.py` - 极简版难度评估工具（推荐常用）
#### 功能描述
最轻量化的任务难度评估脚本，直接输入质检通过的session目录，内存中拼接评估prompt，无任何中间文件，输出仅包含`session-id`和`task_difficulty`两列的精简Excel结果。
#### 核心功能
- ✅ 零中间文件：所有处理在内存中完成，不生成冗余prompt文件
- ✅ 输出极简：结果仅包含session编号和难度等级两列，无额外冗余信息
- ✅ 运行速度最快：省去文件IO开销，适合大批量轨迹评估
- ✅ 支持自定义API地址/密钥：API Key和API地址通过参数传入，无硬编码敏感信息
#### 用法示例
##### 基础用法（使用官方DeepSeek API）
```bash
python batch_deepseek_simple.py --input_dir "D:\sessions\test\质检结果\pass" --api_key "sk-xxxxxxxxx"
```
##### 自定义API地址（兼容第三方DeepSeek兼容接口）
```bash
python batch_deepseek_simple.py --input_dir "D:\sessions\test\质检结果\pass" --api_key "sk-xxxxxxxxx" --api_base "https://自定义API地址/v1"
```
#### 参数说明
| 参数 | 必填 | 说明 |
|------|------|------|
| `--input_dir` | 是 | 质检通过的session目录路径 |
| `--api_key` | 是 | 你的DeepSeek API Key |
| `--api_base` | 否 | DeepSeek API地址，默认值：`https://api.deepseek.com` |
#### 输出说明
- 结果文件自动生成在输入目录同级，文件名为：`session_difficulty_result.xlsx`
- 难度等级范围：`low`（简单）/`medium`（中等）/`high`（复杂）/`xhigh`（高级）/`expert`（专家级）

## ❓ 常见问题
1. **格式转换失败**：检查原始数据格式是否符合对应平台的导出规范，确保输入目录下都是合法的JSON文件
2. **API报错401认证失败**：检查`batch_deepseek_simple.py`中的DeepSeek API Key是否有效，是否还有调用额度
3. **时间筛选不生效**：检查`DEFAULT_AFTER_TIME`格式是否正确，命令行参数是否加双引号包裹
4. **Excel打开乱码**：使用WPS或新版Excel打开，所有脚本默认使用UTF-8编码输出
5. **质检通过率低**：查看`fail`目录下的`failures.csv`错误记录，针对性调整原始数据转换逻辑
