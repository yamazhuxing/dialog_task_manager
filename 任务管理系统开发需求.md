### 背景
我是乙方，现在收到任务，需要通过 Openclaw 和 Hermes 调用 claude opus 4.6 和 claude opus 4.8 进行对话交流，一条样本包含多轮对话，收集并整理对话样本数据給甲方进行大模型蒸馏训练，甲方按照有效样本条数付费。我的目的是交付1000个样本，下发分摊由多个用户执行，最终按多劳多得方式分配酬劳。

### 甲方要求
- 首先，对产生的多轮对话json数据 先通过 convert_openclaw.py 或 convert_hermes.py 进行数据格式转化
- 其次，通过 quality_check.py 进行质量检测，质检通过才是有效样本
- 最后，对pass数据，再使用 batch_deepseek_simple.py 
- run_openclaw_pipeline.py 已经一键完成以上三步，hermes还在开发中，但大体相似
- 样本来源分布：openclaw : hermas ≈ 6 : 4
- model 分布：Claude opus 4.8 : Claude opus 4.6 > 7 : 3（4.8占比可比70%高）
- 场景分布：给定的13类场景至少有一条，且极差比(max/min) < 5（出现最多的场景不多于出现最少的场景的5倍）

### 数据格式
sample目录下，sample\hermes 和 sample\openclaw 存放原始对话数据，sample\openclaw-待质检数据 和 sample\openclaw-待质检数据-质检结果 是经过转化、质量鉴定、难度分级处理后得到的文件夹，其中我已经跑了一个openclaw的例子，最后交付时就是 打包zip压缩 sample\openclaw-待质检数据 和 sample\openclaw-待质检数据-质检结果 即可。

### 任务管理系统开发要求
- 我已经设计好了1200个多轮对话场景，相当于1200个任务，详见 questions_1200.json，用户从平台选择任务，每个任务包含预设多轮提问，用户依次手动复制每个问题到openclaw或hermes进行对话，生成对话文件，每个任务有状态标识，如果已经有人选择或者已完成了，就不能被其他用户选择了，可视化每个用户完成明细
- 每个任务要支持用户提交对应的原始 openclaw或hermes对话文件的入口，对上传文件自动进行格式转化、质量评估、任务难度分级，对于通过的样本进行入库，并更新统计指标
- 针对甲方验收标准，可视化当前指标情况，便于实施跟踪以及调整样本类别，能实时看到整体样本openclaw : model 分布、场景分布、任务难度分布、已通过样本数（交付目标是1000， 如果时间足够，可能会继续增加一些，但整体要满足相关比例）
- 支持多用户登录和同时操作，互不影响
- 希望技术路线简单，尽可能减少依赖，能满足目标就行，UI风格参考 D:\project\daily_stock_analysis，可调整当前代码结构逻辑，实现前后端
- 样本制作很耗时耗钱，所以对于已通过的样本要各位注意保存和备份，避免误删，导致前功尽弃
- 有一个可用的mysql腾讯云数据库，信息详见 .env 可以考虑作为原始数据和验收关键数据的备份，这个需要根据具体情况设计下
- 要支持 对 类似 questions_1200.json 的增加，譬如我后面想增加200个问题，格式还是一样的，希望有入口能支持
- 生成readme文档，包含项目介绍、服务部署等，服务在window开发，最后部署在linux服务器
- 支持对最终质检通过的交付物的zip文件下载
